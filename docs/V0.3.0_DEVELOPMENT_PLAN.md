# DevMan v0.3.0 分阶段开发计划

> 文档更新时间：2026-05-23
> 目标版本：v0.3.0
> 计划策略：先做低风险体验与数据基础，再做配置写入，最后做安装执行

---

## 一、阶段划分

建议将 v0.3.0 拆成三个可验证阶段：

| 阶段 | 目标 | 交付物 |
|---|---|---|
| v0.3.0-alpha | 体验与数据基础 | 快捷键系统、数据采样、Dashboard 趋势卡片 |
| v0.3.0-beta | 配置与策略中心 | 自动清理提醒、镜像源配置、配置备份与恢复 |
| v0.3.0-final | 安装中心 | 安装器检测、白名单包安装、安装进度与扫描联动 |

每个阶段都必须保持：

- `go test ./...` 通过。
- `npm run build` 通过。
- `wails build` 通过。
- 不手动编辑 `frontend/wailsjs/` 和 `frontend/src/bindings/go/`。
- Go JSON tag 继续使用 PascalCase。
- 所有会修改系统状态的操作必须走 Confirm。

## 二、v0.3.0-alpha：体验与数据基础

### 目标

补齐高频操作效率，并为后续自动策略提供可查询的历史数据。

### 任务 A：快捷键系统

范围：

- 保留 `Ctrl+K` 全局搜索。
- 新增 `Ctrl+R` 页面刷新。
- 新增 `Ctrl+1..6` 页面跳转。
- 避免在输入框、textarea、select 中误触发。

重点文件：

- `frontend/src/App.tsx`
- `frontend/src/components/GlobalSearch.tsx`
- `frontend/src/pages/Dashboard.tsx`
- `frontend/src/pages/Cleaner.tsx`
- `frontend/src/pages/Environments.tsx`

验收：

- 快捷键可用且不会破坏输入。
- `Ctrl+R` 在 Dashboard 触发扫描，在 Cleaner 触发分析。
- `npm run build` 通过。

### 任务 B：数据采样模型

后端范围：

- 新增 `MetricSnapshot` 模型。
- 新增 SQLite 表 `metric_snapshots`。
- 新增 registry 方法：`SaveMetricSnapshot` / `ListMetricSnapshots` / `PruneMetricSnapshots`。
- 在 `ScanAll()` 和 `CleanItems()` 成功后记录关键指标。

重点文件：

- `internal/models/models.go`
- `internal/registry/registry.go`
- `internal/registry/registry_test.go`
- `app.go`

验收：

- 扫描后生成环境大小采样。
- 清理后生成释放空间采样。
- 采样表自动创建。
- `go test ./...` 通过。

### 任务 C：Dashboard 趋势卡片

前端范围：

- 新增 API wrapper 调用指标查询方法。
- Dashboard 展示最近趋势。
- 第一版使用轻量 SVG 或 CSS，不强制引入图表库。

重点文件：

- `frontend/src/pages/Dashboard.tsx`
- `frontend/src/devman-types.ts`
- `frontend/src/api/app.ts`
- `app.go`

验收：

- Dashboard 能展示最近扫描趋势。
- 空数据时显示友好占位。
- `npm run build` 通过。

### alpha 完成标准

- 高频导航和刷新可用。
- 历史采样数据已落库。
- Dashboard 有可感知的趋势展示。

## 三、v0.3.0-beta：配置与策略中心

### 目标

让 DevMan 能安全管理工具镜像源，并提供自动清理提醒策略。

### 任务 D：自动清理提醒策略

后端范围：

- 扩展 `AppSettings` 或新增策略模型。
- 新增 `AnalyzeCleanPolicy()` 或等价方法。
- 启动时或 Dashboard 刷新时可返回提醒状态。

前端范围：

- Settings 增加自动清理提醒开关和阈值。
- Dashboard 展示可释放空间提醒。
- 点击提醒跳转 Cleaner。

重点文件：

- `internal/models/models.go`
- `internal/registry/registry.go`
- `app.go`
- `frontend/src/pages/Settings.tsx`
- `frontend/src/pages/Dashboard.tsx`
- `frontend/src/pages/Cleaner.tsx`

验收：

- 默认不自动删除任何文件。
- 超过阈值时有提醒。
- 用户仍需 Confirm 才能清理。

### 任务 E：镜像源配置模型与备份

后端范围：

- 新增 `ConfigProfile` / `ToolConfigBackup` 模型。
- 新增 `config_backups` 表。
- 支持读取当前 npm / pip / cargo / go 配置。
- 应用配置前保存备份。
- npm/pnpm/yarn 第一版只管理用户级配置，不修改项目 `.npmrc`。
- pip 第一版只管理用户级配置，不写入虚拟环境配置。
- cargo 写入 `%USERPROFILE%\.cargo\config.toml` 前必须备份整个文件。
- Go 通过 `go env -w` 写入前必须记录原 `GOPROXY` / `GONOSUMDB`。

重点文件：

- `internal/models/models.go`
- `internal/registry/registry.go`
- `internal/configurator/`（建议新增）
- `app.go`

验收：

- 能读取当前配置。
- 应用前能生成备份。
- 能恢复最近一次备份。
- 写入 history。
- 不默认修改系统 `HTTP_PROXY` / `HTTPS_PROXY` / `NO_PROXY` 环境变量。

### 任务 F：镜像源配置 UI

前端范围：

- Settings 或新增 Config 页面展示工具配置状态。
- 支持选择“官方源 / 国内镜像 / 自定义”。
- 执行前显示变更预览和 Confirm。
- 支持恢复上次配置。

重点文件：

- `frontend/src/App.tsx`
- `frontend/src/components/Sidebar.tsx`
- `frontend/src/pages/Settings.tsx`
- `frontend/src/pages/ConfigCenter.tsx`（建议新增）
- `frontend/src/devman-types.ts`

验收：

- 用户能明确看到将修改哪些工具。
- 失败时 Toast 明确显示工具和原因。
- 恢复操作可取消。

### beta 完成标准

- 自动清理只提醒不删除。
- 镜像源配置有预览、备份、恢复。
- 所有配置写入都可追踪。

## 四、v0.3.0-final：安装中心

### 目标

提供安全、白名单化的一键安装能力，并与现有扫描能力联动。

### 任务 G：安装器检测

后端范围：

- 新增 `internal/installer/` 包。
- 检测 winget / scoop / choco 是否可用。
- 获取版本信息。
- 判断是否可能需要管理员权限。
- 检测命令使用 `winget --version` / `scoop --version` / `choco --version`。
- 检测结果可缓存，但必须允许手动刷新。

重点文件：

- `internal/installer/installer.go`
- `internal/installer/winget.go`
- `internal/installer/scoop.go`
- `internal/installer/choco.go`
- `app.go`

验收：

- 可显示本机可用安装器。
- 检测失败不影响应用启动。
- 命令执行有超时。
- 能提示 scoop 通常为用户级安装、choco 通常需要管理员权限、winget 可能触发 UAC 或安装器交互。

### 任务 H：白名单包清单

后端范围：

- 内置 `InstallablePackage` 列表。
- 前端只能传 `PackageKey`，不能传任意命令或包名。
- 后端根据白名单生成命令预览。
- winget 命令必须使用精确 ID 和固定 source，例如 `winget install --id <id> --exact --source winget`。
- scoop/choco 包名也必须来自后端白名单。

建议首批包：

- Git
- Node.js LTS
- Go
- Python
- Docker Desktop
- VS Code

验收：

- 包清单可展示。
- 生成命令只来自后端白名单。
- 非法 PackageKey 被拒绝。
- 命令预览必须展示安装器、包 ID、source、是否可能需要管理员权限。

### 任务 I：安装执行与进度事件

后端范围：

- 新增 `InstallPackage(packageKey, manager)` API。
- 执行前写入 history pending 状态。
- 通过 Wails event emit 安装日志和状态。
- 安装完成后触发或建议 `ScanAll()`。
- 执行使用 `exec.CommandContext` 和超时控制。
- 安装日志按 stdout/stderr 流式推送，但最终结果仍由 API 返回。

前端范围：

- 新增 Install 页面或 Config Center 中的安装 tab。
- 显示命令预览。
- Confirm 后执行。
- 实时显示日志和完成状态。

重点文件：

- `internal/installer/`
- `app.go`
- `frontend/src/pages/InstallCenter.tsx`（建议新增）
- `frontend/src/api/app.ts`
- `frontend/src/devman-types.ts`

验收：

- 安装前可取消。
- 安装日志可见。
- 成功/失败有 Toast。
- 完成后扫描结果可刷新。

### final 完成标准

- v0.3.0 所有新增能力都有可见入口。
- 安装、配置、清理三类写操作都有确认、日志和历史记录。
- `go test ./...`、`npm run build`、`wails build` 通过。

## 六、风险与约束

1. **禁止任意命令执行**：前端只能传 key，后端生成命令。
2. **禁止静默删除**：自动清理只能提醒，清理必须 Confirm。
3. **配置必须可恢复**：镜像源和代理写入前必须备份。
4. **安装必须白名单化**：不接受用户输入的包 ID 直接执行。
5. **事件不替代返回值**：安装和配置进度事件是增强，API 仍需返回最终结果。
6. **继续保持 PascalCase JSON tags**。
7. **不要手动编辑 Wails 生成文件**。
