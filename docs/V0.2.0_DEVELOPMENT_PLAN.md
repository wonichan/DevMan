# DevMan v0.2.0 分阶段开发计划

> 文档更新时间：2026-05-23
> 目标版本：v0.2.0
> 计划策略：先补基础体验，再扩展扫描/清理，最后做高级交互

---

## 一、阶段划分

建议将 v0.2.0 拆成三个可验证阶段：

| 阶段 | 目标 | 交付物 |
|---|---|---|
| v0.2.0-alpha | 体验基础补齐 | Toast 全接入、Confirm 统一、Settings 持久化 |
| v0.2.0-beta | 扫描与清理升级 | 自定义扫描路径、新增扫描器、Cleaner 深度扫描 |
| v0.2.0-final | 高级体验完善 | 全局搜索、主题设置基础、迁移进度实时显示 |

每个阶段都必须保持：

- `go test ./...` 通过。
- `npm run build` 通过。
- `wails build` 通过。
- 不编辑 `frontend/wailsjs/` 和 `frontend/src/bindings/go/` 中的生成文件。
- Go JSON tag 继续使用 PascalCase。

---

## 二、v0.2.0-alpha：体验基础补齐

### 目标

让现有功能具备稳定、可见、可持久化的用户反馈和设置能力。

### 任务 A：Toast 全接入

范围：

- 替换业务页面中用户可见错误的 `console.error`。
- 保留必要的开发调试日志，但不能作为唯一错误反馈。

重点文件：

- `frontend/src/pages/Dashboard.tsx`
- `frontend/src/pages/Environments.tsx`
- `frontend/src/pages/Migration.tsx`
- `frontend/src/pages/Versions.tsx`
- `frontend/src/pages/Cleaner.tsx`
- `frontend/src/hooks/useToast.ts`

验收：

- 所有加载失败、刷新失败、迁移失败、清理失败都有 Toast。
- 成功操作有简短成功 Toast。
- `npm run build` 通过。

### 任务 B：Confirm 统一

范围：

- 迁移、清理、重置、导入覆盖等危险操作统一使用 `useConfirm`。
- 不新增多个确认弹窗实现。

重点文件：

- `frontend/src/components/ui/ConfirmDialog.tsx`
- `frontend/src/pages/Migration.tsx`
- `frontend/src/pages/Cleaner.tsx`
- `frontend/src/pages/Settings.tsx`

验收：

- 危险操作执行前都能取消。
- ESC 可取消确认弹窗。
- 焦点返回触发元素。

### 任务 C：Settings 持久化

后端范围：

- 新增 `models.AppSettings`。
- 在 `registry.migrate()` 中新增 `settings` 表。
- 新增 settings 读写方法。
- 在 `app.go` 暴露 `GetSettings()`、`SaveSettings(settings)`。

前端范围：

- `Settings.tsx` 从后端加载设置。
- 保存设置时写入 SQLite。
- 保存成功/失败接入 Toast。

重点文件：

- `internal/models/models.go`
- `internal/registry/registry.go`
- `internal/registry/registry_test.go`
- `app.go`
- `frontend/src/devman-types.ts`
- `frontend/src/pages/Settings.tsx`

验收：

- 修改设置后重启应用仍保留。
- settings 表可自动创建。
- 未保存时不应误导用户已持久化。
- `go test ./...`、`npm run build`、`wails build` 通过。

### alpha 完成标准

- Settings 页面不再只有本地状态。
- 用户操作反馈完整。
- v0.2.0 后续功能可以复用 settings / toast / confirm。

## 三、v0.2.0-beta：扫描与清理升级

### 目标

提升 DevMan 对真实 Windows 开发环境的识别能力，并让 Cleaner 能发现有价值的大目录。

### 任务 D：自定义扫描路径

后端范围：

- 增加 `ScanOptions` 或等价结构。
- `ScanAll()` 读取 settings 中的 `CustomScanPaths`。
- 扫描器在默认路径之外追加自定义路径检测。

前端范围：

- Settings 页面增加自定义路径列表。
- 支持添加、删除路径。
- 路径保存到 settings。

重点文件：

- `internal/scanner/scanner.go`
- `internal/scanner/common.go`
- `internal/scanner/nodejs.go`
- `app.go`
- `frontend/src/pages/Settings.tsx`

验收：

- 添加非标准路径后，重新扫描可识别环境。
- 无效路径不会中断扫描。
- 自定义路径可持久化。

### 任务 E：新增扫描器

建议顺序：

1. Docker
2. pnpm / yarn / bun
3. VS Code / JetBrains

后端范围：

- 新增 scanner 文件。
- 注册到 `NewEngine()`。
- 增加 `modelsForScanner()` metadata。
- 增加测试。

重点文件：

- `internal/scanner/docker.go`
- `internal/scanner/javascript_tools.go`
- `internal/scanner/scanner.go`
- `internal/scanner/common_test.go`

验收：

- 新工具可出现在 Dashboard / Environments。
- 版本命令真实执行，有超时。
- 常见缓存路径写入 `EnvPath`。

### 任务 F：Cleaner 深度扫描

后端范围：

- 扩展 `AnalyzeCleanable()` 的目录识别。
- 区分缓存类和项目依赖/构建产物。
- 清理时仅删除用户选中的项目。

前端范围：

- Cleaner 页面展示风险或类别。
- 默认选中缓存类，不默认选中项目依赖类。
- 清理确认中展示数量和总大小。

重点文件：

- `app.go`
- `internal/models/models.go`
- `frontend/src/pages/Cleaner.tsx`
- `frontend/src/devman-types.ts`

验收：

- 可识别 npm/pnpm/yarn/pip/gradle/cargo/go 缓存。
- `node_modules` 和 `target` 不默认选中。
- 清理后重新分析结果更新。
- 删除失败有 Toast 错误提示。

### beta 完成标准

- 扫描范围明显扩大。
- Cleaner 对大目录有实际价值。
- 不误删未选择目录。

## 四、v0.2.0-final：高级体验完善

### 目标

补齐快速导航、主题策略和迁移过程可视化，让 v0.2.0 达到可发布状态。

### 任务 G：全局搜索

前端范围：

- 新增全局搜索组件。
- 支持页面、环境、操作搜索。
- `App.tsx` 提供统一导航入口。

重点文件：

- `frontend/src/App.tsx`
- `frontend/src/components/GlobalSearch.tsx`
- `frontend/src/components/Sidebar.tsx`
- `frontend/src/pages/Environments.tsx`

验收：

- 可通过顶部入口打开搜索。
- 可搜索并跳转页面。
- 可搜索环境并跳转到相关页面。

### 任务 H：主题设置基础

范围：

- Settings 中保存主题策略：`dark` / `system`。
- 不强制完成完整浅色主题。
- 保持当前深色主题稳定。

重点文件：

- `frontend/src/App.tsx`
- `frontend/src/pages/Settings.tsx`
- `frontend/src/style.css`
- `tailwind.config.js`

验收：

- 主题设置可保存。
- 当前 UI 不退化。
- 为后续浅色主题留出结构。

### 任务 I：迁移进度实时显示

后端范围：

- migrator 在关键步骤 emit progress event。
- 保留现有 `Migrate()` 返回结果。
- 第一版按步骤百分比，不强求精确字节级进度。

前端范围：

- Migration 页面订阅 progress event。
- 展示当前步骤、日志和百分比。
- 失败时展示最后状态和错误。

重点文件：

- `internal/migrator/migrator.go`
- `app.go`
- `frontend/src/pages/Migration.tsx`

验收：

- 迁移过程中 UI 能更新进度。
- 迁移结束后显示成功/失败结果。
- 旧的迁移结果逻辑仍可用。

### final 完成标准

- v0.2.0 所有计划功能可被用户感知。
- 构建、测试、基础手动 QA 通过。
- `docs/STATUS.md` 更新为 v0.2.0 状态。

## 六、风险与约束

1. **不要编辑生成文件**：`frontend/wailsjs/` 和 `frontend/src/bindings/go/` 由 Wails 生成。
2. **JSON 字段保持 PascalCase**：Wails 桥接依赖现有命名。
3. **Cleaner 删除风险高**：默认选择必须保守，危险目录必须明确确认。
4. **迁移进度不要打断现有流程**：事件机制应作为增强，不破坏 `Migrate()` 返回值。
5. **主题切换成本高**：v0.2.0 先做设置结构，完整浅色模式后续单独规划。
